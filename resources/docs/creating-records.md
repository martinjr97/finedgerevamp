# Creating Records

This section covers the most common record-creation workflows.

## Admin: Creating Core Records

Typical records created by admins include:

- users and roles
- companies and branches
- customers and customer groups
- loan products and rates
- loan applications and approvals

General pattern:

1. Open the module from the sidebar.
2. Click **Create** (or equivalent action button).
3. Complete required fields.
4. Save and confirm the success message.
5. Review the new record in list or detail view.

## Customer: Creating Loan Requests

Customer users create records mostly through guided flows:

1. Start loan or collateral loan flow from dashboard.
2. Select channel and enter required details.
3. Run calculation and confirm terms.
4. Submit the application.
5. Track status from dashboard and statement pages.

## Validation and Errors

If validation fails:

- required fields are highlighted
- errors appear at the top of the page
- no record is saved until all required values are valid
